# 🤖 RouterAi — Telegram-бот для бесплатных нейросетей

RouterAi — это асинхронный Telegram-бот на базе `aiogram 3`, который автоматически находит актуальные бесплатные модели на OpenRouter и позволяет общаться с ними прямо в чате. Бот поддерживает стриминг, системные промты, шифрование данных и удобное управление разговорами.

## ✨ Ключевые особенности

- **Временные разговоры по умолчанию**: история существует только в памяти. Чтобы сохранить диалог, используйте `/save`. Команда `/new` мгновенно стирает текущую сессию.
- **Умный выбор моделей**: меню разделено на категории (💬 Текстовые, 👁 Мультимодальные, 🎵 Музыка/Аудио, 🖼 Картинки, 📦 Прочие). Сначала выбирается категория, затем модель. Vision-модели (распознают фото) помечены 👁. Модели сортируются по аптайму бесплатного эндпоинта с цветовой индикацией (🟢 ≥95%, 🟡 80–95%, 🔴 <80%, ⚪ нет данных).
- **Безопасность данных**: пользовательские ключи OpenRouter и тексты сохранённых сообщений шифруются алгоритмом Fernet (AES-128-CBC + HMAC). Файл БД создаётся с правами `600`.
- **Продвинутое управление**: постоянная нижняя клавиатура для быстрого доступа, пресеты температуры, пошаговый гайд по добавлению своего ключа API.

## 📋 Команды бота

| Команда | Описание |
|---|---|
| `/start` | Приветствие, начинает новый временный разговор |
| `/help` | Справка по командам и возможностям |
| `/new` | Начать новый временный разговор (старый сразу стирается без подтверждения) |
| `/save <название>` | Сохранить текущий разговор в именованный чат (без названия — автоимя; повторный `/save` обновляет загруженный чат) |
| `/saved` или `/chats` | Список сохранённых чатов: загрузить (кнопка) или удалить (🗑) |
| `/reset` | Очистить историю текущего разговора |
| `/password <пароль>` | Шифровать сохранённые чаты своим паролем 🔒 (zero-knowledge; `/password -` сбросить) |
| `/models` или `/model` | Выбрать модель: сначала категория, затем модель (👁 = vision; сортировка по аптайму) |
| `/system <текст>` | Установить системный промт (`/system -` очистить) |
| `/temp <0–2>` | Установить температуру генерации |
| `/setkey <ключ>` | Ввести свой ключ OpenRouter (`/setkey -` сбросить на общий; сообщение с ключом автоматически удаляется) |

## ⌨️ Постоянная клавиатура

Для удобства в чате закреплена нижняя клавиатура:
- **🤖 Модели** — быстрый доступ к выбору модели.
- **💬 Сохранённые** — список ваших чатов.
- **➕ Новый чат** — очищает память и начинает новый временный диалог.
- **💾 Сохранить** — сохраняет текущий диалог.
- **⚙️ Настройки** — инлайн-меню с пресетами температуры (0.3 / 0.7 / 1.0 / 1.3) и кнопкой «Свой ключ» для пошагового гайда по добавлению ключа OpenRouter.
- **ℹ️ Помощь** — вызов справки.

## 🛠 Как это работает

1. **Поиск моделей**: При запросе списка моделей бот обращается к `GET /api/v1/models`, фильтрует бесплатные (суффикс `:free` или цена 0) и кэширует их с настраиваемым TTL.
2. **Обогащение аптаймом**: Для каждой модели параллельно (с семафором на 6 одновременных запросов)_fetchится `GET /api/v1/models/{id}/endpoints`. Из ответа берётся `uptime_last_1d` для определения индикатора (🟢, 🟡, 🔴, ⚪) и сортировки.
3. **Классификация**: Модели раскладываются по категориям на основе `input_modalities`/`output_modalities` (текстовые, мультимодальные, аудио, картинки, прочие); vision определяется по наличию `image` во входных модальностях.
4. **Общение**: Сообщения пользователя отправляются в `POST /api/v1/chat/completions` со стримингом (SSE).
5. **Сессии**: Текущий разговор живёт исключительно в оперативной памяти (`session.py`) и теряется при перезапуске бота, пока не будет сохранён через `/save`. Ответ модели стримится в одно сообщение (редактируется не чаще раза в ~1.3 секунды). Длинные ответы автоматически режутся на части по 4096 символов.

## 📁 Структура проекта

```text
orbot/
├── bot/
│   ├── main.py            # точка входа, polling
│   ├── config.py          # загрузка .env (BOT_TOKEN, OPENROUTER_API_KEY, ENCRYPTION_KEY, ...)
│   ├── crypto.py          # шифрование Fernet (ключи и сообщения at rest)
│   ├── db.py              # SQLite (aiosqlite): users, chats, messages; шифрование, права 600
│   ├── session.py         # временные in-memory сессии разговора
│   ├── openrouter.py      # async-клиент OpenRouter (модели + аптайм + стрим)
│   ├── models_cache.py    # кэш free-моделей с TTL, обогащение аптаймом, категории/vision
│   ├── runtime.py         # общие синглтоны
│   ├── keyboards.py       # постоянная нижняя клавиатура
│   ├── utils.py           # резка >4096, экранирование MarkdownV2, обрезка истории
│   └── handlers/
│       ├── chats.py       # /start /new /save /saved /reset, загрузка/удаление сохранённых
│       ├── settings.py    # /models (вкладки), /system /temp /setkey, инлайн-настройки
│       ├── dialog.py      # текст → стрим ответа модели (в память, без записи в БД)
│       └── common.py      # общие помощники (active_session)
├── requirements.txt
├── .env.example
└── README.md
```

## 🚀 Установка и запуск

1. Убедитесь, что установлен **Python 3.11+** (разрабатывалось и тестировалось на 3.13).
2. Клонируйте репозиторий и перейдите в его папку.
3. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Для Windows: .venv\Scripts\activate
   ```
4. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
5. Скопируйте пример конфигурации и заполните его:
   ```bash
   cp .env.example .env
   ```
6. Запустите бота из корневой папки проекта:
   ```bash
   python -m bot.main
   ```

## ⚙️ Переменные окружения (.env)

| Переменная | Описание | Значение по умолчанию |
|---|---|---|
| `BOT_TOKEN` | **Обязательно**. Токен вашего Telegram-бота (получить у @BotFather). | - |
| `OPENROUTER_API_KEY` | **Обязательно**. Общий ключ API OpenRouter по умолчанию (получить на openrouter.ai). | - |
| `ENCRYPTION_KEY` | **Обязательно**. Секретный ключ для шифрования данных в БД (Fernet). Сгенерировать: `python -c "import secrets; print(secrets.token_urlsafe(48))"`. **Внимание:** при смене ключа старые зашифрованные данные станут нечитаемы! | - |
| `DB_PATH` | Путь к файлу базы данных SQLite. | `orbot.db` |
| `MODELS_CACHE_TTL_HOURS` | Время жизни кэша списка бесплатных моделей в часах. | `6` |
| `HISTORY_MAX_MESSAGES` | Максимальное количество сообщений в контексте, отправляемых в API. | `20` |
| `DEFAULT_MODEL` | Идентификатор модели, используемой по умолчанию. | `deepseek/deepseek-r1:free` |

## 🔒 Безопасность и приватность

- Все чувствительные данные (личные ключи OpenRouter пользователей и тексты сохранённых сообщений) шифруются на уровне приложения с помощью симметричной криптографии **Fernet** (AES-128-CBC + HMAC).
- **Приватные чаты под паролем (zero-knowledge)**: команда `/password <пароль>` включает шифрование сохранённых чатов ключом, выведенным из пароля пользователя через **scrypt**. Пароль нигде не хранится (только в памяти процесса). Такой чат (помечен 🔒) невозможно прочитать без пароля — **даже владельцу сервера с полным доступом к БД и `ENCRYPTION_KEY`**. Чтобы открыть его после перезапуска бота, нужно снова ввести пароль через `/password`. Забытый пароль означает безвозвратную потерю чата.
- **Политика no-logs**: содержимое переписок нигде не пишется в логи. Логирование HTTP-клиента и событий aiogram приглушено, чтобы тела сообщений не попадали в журналы, даже когда данные проходят через сервер к OpenRouter.
- По умолчанию разговоры временные — на диске их нет вообще, пока пользователь сам не вызовет `/save`.
- Файл базы данных `orbot.db` создаётся с правами `600` (доступен только владельцу).
- Сообщения, содержащие API-ключ (`/setkey`) или пароль (`/password`), немедленно удаляются из чата.
- Файл `.env` обязательно должен быть в `.gitignore`.
- **Оговорка**: секрет `ENCRYPTION_KEY` хранится в `.env` в открытом виде — он защищает чаты, сохранённые *без* пароля, от кражи дампа БД; чаты под паролем он не раскрывает. Надёжно защищайте сервер.
- **Совет по ключу**: при создании ключа на OpenRouter задайте в поле «Credit limit» небольшой лимит (например `0.001$`) для контроля расходов.

## ⚠️ Ограничения

- Бесплатные модели OpenRouter имеют жёсткие rate-limits и иногда бывают перегружены. Если бот вернёт ошибку, просто попробуйте ещё раз или переключитесь на другую модель.
- Список доступных бесплатных моделей динамический и постоянно меняется со стороны OpenRouter.
- Аудио-модели пока отображаются в боте исключительно для ознакомления. Для их использования в чате требуется отдельный API-endpoint, который на данный момент не реализован.
